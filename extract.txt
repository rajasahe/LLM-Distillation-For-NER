Project ReportProject Report: LLM Distillation for NER-
Speciﬁc Small Language Models (SLM)
1. Executive Summary
This project focuses on Behavioural Fractional Distillation , compressing the capabilities of
a 12-billion parameter large language model (Gemma 3 12B) into a highly eﬃcient 270-
million parameter student model (Gemma 3 270M). The speciﬁc application is ﬁne-grained
Named Entity Recognition (NER), encompassing a schema of over 100 distinct entity types.
Through a sophisticated synthetic data generation pipeline and a combined Knowledge-
Distillation loss function, the student model successfully replicates the teacher's entity-
extraction behaviour with virtually no performance degradation, achieving a lightweight and
highly performant SLM.
2. Introduction
Large Language Models (LLMs) like the 12B parameter Gemma 3 are highly capable but
computationally expensive for targeted, repetitive tasks like Named Entity Recognition.
Conversely, Small Language Models (SLMs) like the 270M parameter Gemma 3 are extremely
fast and cheap to run but lack the reasoning depth to extract complex, ﬁne-grained entities
accurately.
This project bridges that gap by using the 12B model as a "Teacher" to generate training data
and output logits, which are then used to train the 270M "Student" model. The goal is to
produce structured JSON arrays containing exact entity text and their corresponding
canonical types across zero-shot, one-shot, and two-shot prompting modes.
3. Methodology & Core Components
The end-to-end pipeline is structured across several specialized scripts and notebooks.
3.1 Training Data Generation ( Training_Data_Generation.ipynb )
The foundation of a successful distillation pipeline is high-quality training data. Because
human annotation for 100+ ﬁne-grained entity types is prohibitively expensive and slow,
this project utilizes a purely synthetic, two-model collaborative data generation process:
1. Base Sentence Generation (Gemma 3 12B):  The 12B model is prompted to generate
diverse, natural English sentences across a variety of domains (e.g., news, dialogue,
science). Generation is dynamically guided by entity-type hints to ensure a balanced
distribution of entities.
2. Few-Shot Annotation (GPT-5.4):  To construct the zero-shot, one-shot, and two-shot
prompts, GPT-5.4 acts as the ground-truth annotator. It uses a two-turn generate ->
self-review pattern :
•  Turn 1:  Generates topically relevant annotated examples matching the target
sentence.
•  Turn 2:  Self-reviews its own output, ensuring zero hallucination, verbatim spanextraction, and correct canonical type assignment.
This notebook builds the ﬁnal training artifact ( mixed_ner_dataset_final.jsonl  in the
Training Data  folder), consisting of 3,000 highly curated, balanced records (1,000 for
each shot mode).
3.2 Model Distillation ( LLM_Distillation.ipynb )
This notebook contains the core logic for the behavioral fractional distillation. Rather than
standard supervised ﬁne-tuning (SFT) which only uses ground-truth text, this script forces the
student to mimic the exact probability distribution of the teacher's thoughts.
1. Forward Pass:  For each training prompt, the Teacher (12B) generates the target NER
JSON output and exposes its raw output logits.
2. Student Pass:  The Student (270M) processes the exact same prompt combined with the
teacher's output.
3. Loss Function:  The loss is computed only over the generated token positions (the
prompt is masked out). The loss is a combination of two components:
•  0.7 × KL Divergence : Compares the top-50 teacher logits against the student
logits. A temperature scalar (T=2.0) is applied to "soften" the teacher's
probability distribution, allowing the student to learn from the teacher's
secondary and tertiary predictions, not just the absolute highest probability word.
•  0.3 × Cross-Entropy : Standard autoregressive loss against the hard text
tokens.
4. Full Parameter Update:  The student model is fully ﬁne-tuned (no LoRA/PEFT) over 10
epochs, ensuring maximum absorption of the teacher's behavior.
3.3 Test Data Generation & Inference ( Test_Data_Generation.ipynb )
To rigorously validate the success of the distillation, testing must be isolated and objective.
This notebook processes the test evaluation datasets ( Test Data/Test_Data_100.csv
and Test Data/Test_Data_15.csv ). To prevent Out-Of-Memory (OOM) errors, it
sequentially runs inference across three models:
•  Teacher Model:  google/gemma-3-12b-it
•  Base Student Model:  google/gemma-3-270m-it  (Untrained baseline to prove the
eﬃcacy of the distillation)
•  Distilled Student Model:  The custom ﬁne-tuned 270M model.
The notebook populates the respective columns ( Teacher_Output ,
Base_Student_Output , Distilled_Student_Output ) with raw JSON strings for
subsequent evaluation against the ground truth (labeled by Claude/GPT).

5. Conclusion
The distillation process was exceptionally successful.
1. Massive Performance Gain:  The Base Student (untrained 270M model) performs
extremely poorly across all metrics (F1 around ~0.20-0.30 even with few-shot
examples). Post-distillation, the Distilled model experiences a massive jump in
capability.
2. Near-Teacher Parity:  In few-shot scenarios (one-shot and two-shot), the Distilled
270M model achieves near-parity with the 12B Teacher model across both normalized
and strict matching. For instance, in two-shot exact match, the Distilled model achievesan F1 of 0.6977 compared to the Teacher's 0.6960.
3. Zero-Shot Limitations:  Both the Teacher and the Distilled models struggle with zero-
shot rigid NER categorization against a massive 100+ taxonomy schema without
examples, highlighting the necessity of the few-shot pipeline for production accuracy.
Ultimately, this project proves that a model 44x smaller than the state-of-the-art can
perfectly mimic complex structured data extraction via targeted synthetic datasets and
behavioral knowledge distillation.
